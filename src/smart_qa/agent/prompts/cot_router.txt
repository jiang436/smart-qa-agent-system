## 思维链：意图分类

请一步步推理用户的意图，不要直接跳结论。

### 步骤1: 理解用户说什么
- 提取用户问题的关键词
- 判断用户是在描述问题、询问信息、还是下达指令

### 步骤2: 判断意图类型（关键区分）

- **qa**: 询问「怎么做」「是什么」「能不能」「有什么功能」等操作方法或规格参数
  - 示例: "怎么设置定时清扫？" "滤网怎么换？" "边刷怎么拆卸？"
  - 要点: 用户想知道**操作步骤**，不是问价格型号

- **troubleshoot**: 报告设备异常、故障、不工作、错误码
  - 示例: "机器不工作了" "显示错误码E05" "一直卡在门槛" "扫地机器人不工作了怎么办"
  - 要点: 用户报告设备**出了问题/坏了/异常**，即使后面带"怎么办"也是故障排查

- **consumables**: 询问「买什么」「多少钱」「什么型号」「哪里买」等购买或推荐
  - 示例: "边刷该换了买什么型号？" "滤网多少钱？" "拖布哪里买？"
  - 要点: 用户想知道**型号/价格/渠道**，想买东西

- **device_control**: 下达设备操作指令，包括开始/停止清扫、回充、切换模式、查询状态、设置定时
  - 示例: "开始清扫" "停止" "回去充电" "设为安静模式" "检查我的设备状态" "设置每天9点清扫"
  - 要点: 用户想**控制设备**做某事，不是问问题

- **report**: 要求查看使用统计、生成报告、查看清洁记录或耗材状态
  - 示例: "生成我的使用报告" "查看使用统计" "给我一份清洁周报" "看看清洁数据"
  - 要点: 用户想看**数据汇总和统计**

- **general**: 闲聊、问候、与扫地机完全无关的内容
  - 示例: "你好" "今天天气不错"

### 步骤3: 关键区分
- 设备"不工作/坏了/故障/异常" → **troubleshoot**（即使带"怎么办"也属于故障排查）
- "XX怎么换/如何更换/怎么安装" → **qa**（操作步骤）
- "XX多少钱/买什么型号/哪里买" → **consumables**（购买推荐）
- "开始XX/停止XX/设置XX" → **device_control**（设备控制）
- "查看/生成/我的XX报告/统计" → **report**（报告查询）
- 同时包含"换"和"型号/价格" → 看主要意图：问"怎么换"是qa，问"买什么"是consumables

### 步骤4: 输出
仅输出一个词: qa / troubleshoot / consumables / device_control / report / general
